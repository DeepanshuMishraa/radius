## 1. Architecture Review

### Issue 1 — Electrobun IPC: Use defineRPC (RESOLVED)
Electrobun provides typed RPC, not generic IPC. The design doc described a hand-rolled IPC bridge. Updated to use `BrowserView.defineRPC` with shared types in `src/shared/types.ts`.

### Issue 2 — OAuth redirect capture (ACCEPTED RISK)
Highest-risk unknown. BrowserWindow navigation interception for OAuth redirect is unverified in Electrobun. User chose to build assuming it works and fix if broken.

### Issue 3 — Zero tests in plan (DEFERRED)
User chose to skip tests for v1, add in v2. Documented as known risk.

## 2. Code Quality Review

Tests deferred to v2. Coverage diagram shows 30+ untested codepaths across auth, sync, db, gmail, and IPC modules.

## 4. Performance Review

- `bun:sqlite` synchronous blocking during full sync — `Bun.sleep(0)` is unverified mitigation
- IPC `getInbox(limit, offset)` has no upper bound on `limit` — cap at 100

## Critical Gaps (3)

1. **OAuth redirect capture** — no test, no fallback, would hang silently if BrowserWindow can't intercept navigation
2. **IPC `limit` unbounded** — no cap, could cause OOM on large `limit` values
3. **DOMPurify in Bun** — unverified, fallback to `isomorphic-dompurify` planned but not tested

## Failure Modes

| Codepath | Failure | Test? | Handling? | User sees |
|----------|---------|-------|-----------|-----------|
| OAuth redirect | BrowserWindow can't intercept | No | No fallback | Auth hangs |
| SQLite blocking | 10s UI freeze during sync | No | Bun.sleep(0) | Frozen progress bar |
| Gmail 429 | Rate limited | No | Exponential backoff | Paused progress |
| History gap | history.list 404 | No | Full re-sync | Slow re-sync |
| Token refresh | Refresh token revoked | No | Re-prompt OAuth | Auth screen |
| DOMPurify load | jsdom shim fails | No | isomorphic-dompurify | Sanitization works |
| IPC limit | Request 100k messages | No | No cap | Memory crash |
